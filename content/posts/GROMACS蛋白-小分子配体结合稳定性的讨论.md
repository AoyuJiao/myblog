小分子配体或药物如何结合到蛋白的活性口袋上，一直是结构生物学和药理学津津乐道的话题。基于湿实验测定的方法，包括X射线晶体衍射法等，尽管直观可靠，但毕竟成本高昂、操作复杂，只能用于在确认药效后的实证而非一个探索手段。为解决此问题，计算机辅助药物设计（CADD）应运而生。得益于计算机计算能力的迅猛提升，通过模拟计算来研究结合过程和模式、药物设计、甚至面向整个小分子库的活性小分子挖掘成为可能。

CADD挖掘先导化合物的方法之一便是分子对接，其简单、高效，应用十分广泛。但其存在一个致命的缺点，即分子对接只能研究一个静态的结合过程，不能确认其结合的稳定性如何。分子动力学模拟可用于在一定程度上解决此问题，故本文我们讨论如何基于GROMACS的模拟结果了解蛋白-小分子配体结合的稳定性。

# 均方根偏差（RMSD）
RMSD是一个数学概念，用来衡量两组值之间的平均相对偏差，且通过平方再开方的方式避开正负相消的问题。对于原子体系$A$，其在第$n$帧的RMSD可被描述为：
$$
RMSD_{A,A_{0}}=\sqrt{\frac{\sum_{n=1}^{N}{(A_0 - \overline{A_{n}})^2} }{N} } 
$$
其中，$A_n$指的是第$n$帧结构原子位置。$A_0$指的是第0帧也就是初始结构的原子位置，N代表总帧数。这实质上是在比较此帧结构相对与初始结构，在原子坐标（几何）上的偏差。如果对每一帧都计算一次RMSD，并作为一个数据点绘制在以帧号（时间）为横轴，以RMSD值为纵轴的散点图中，在数据点足够多的情况下，就可以得到RMSD相关于时间的曲线。此曲线是分析蛋白-小分子配体结合稳定性最重要的方法。

## RMSD的计算方法
GROMACS内置了gmx rms命令来完成此计算。假设在工作目录下有`md.txc`和`md.tpr`两个文件，则可在终端做如下操作计算。

首先，推荐新建一个目录，用来储存RMSD数据，以及为后续绘图做准备，可避免文件混乱。
```
mkdir rmsd && cd rmsd
```
接下来，计算蛋白自体RMSD。此处建议基于蛋白碳骨架（`C-alpha`）而非整个蛋白（`Protein`）进行计算，从而减少侧链对结果的干扰。选择蛋白碳骨架（`C-alpha`）进行输出。
```
gmx rms -f ../md.xtc -s ../md.tpr -o rmsd_protein.xvg
```
接下来，计算配体相对于蛋白的RMSD[^1]。此处同样建议基于蛋白碳骨架（`C-alpha`）计算，选择小分子组进行输出。
```
gmx rms -f ../md.xtc -s ../md.tpr -o rmsd_lig.xvg
```
输出的`.xvg`文件是一个同时包含数据和可视化方案的文件形式，当使用XmGrace等工具打开时，可以直接读取数据并完成绘图。
![小分子RMSD](/images/小分子RMSD.webp)
上图即为一段由XmGrace绘制的小分子配体相对于蛋白骨架碳原子的RMSD曲线，整体波动在0.5nm左右，并不太大，但又几处明显的波动。此时，可以参考蛋白骨架碳原子的自体RMSD曲线。
![蛋白RMSD](/images/蛋白RMSD.webp)
如图为蛋白骨架碳原子的自体RMSD曲线，可以看到蛋白的RMSD约为0.25nm,且相对平稳，没有明显的跃变，说明小分子的RMSD波动并非由蛋白引起，而源于自身的构象改变。后续可根据轨迹分析进行确认。`.xvg`文件是纯文本型的，有标准的数据输出格式，可以复制出来进行后续的高阶统计分析与可视化。

# 均方根波动（RMSF）
RMSF与RMSD本质上属于同一种计算方法，但RMSD面向所关注的整个体系，而RMSF可以面向一个残基甚至一个原子，因此常用来描述口袋的刚柔性。一般来说，一个残基的RMSF越大，其柔性越高。

## RMSF的计算

首先，推荐新建一个目录，用来储存RMSD数据，以及为后续绘图做准备，可避免文件混乱。
```
mkdir rmsf && cd rmsf
```
市面上的教程常推荐研究整个蛋白中所有原子的RMSF,这是从蛋白质性质研究中遗传下来的做法，但对于验证配体在蛋白口袋中的结合稳定性意义不大。如要研究蛋白口袋的刚柔性，只把口袋中的残基纳入计算即可。因此，我们需要通过以下命令定义“选择属于`Protein`组的所有原子，并且这些原子距离被重命名为`MOL`的分子在 5 埃（Å）[^2]的范围之内”，如果小分子不是`MOL`，请在命令中对应替换。
```
gmx select -s ../md.tpr -select "group Protein and within 0.5 of resname MOL" -on clean_site.ndx
```
